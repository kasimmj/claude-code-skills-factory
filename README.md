## 🏭 What is it?

Writing a good Claude Code skill is hard. You need:
- The right frontmatter format (`name`, `description`, `triggers`)
- Specific, unambiguous trigger phrases
- A "When NOT to use" section
- Clear failure modes
- Concrete examples

**This tool does all of that automatically.** You describe what you want in plain English (or Arabic, French, Spanish). It generates a lint-passing, idiomatic, ready-to-install skill.

---

## ⚡ Quick Start

```bash
npm i -g claude-code-skills-factory
```

```bash
$ claude-skill new "I want a skill that audits my Tailwind classes and removes duplicates"

✨ Generating skill...

  ✓ Name:        tailwind-deduper
  ✓ Triggers:    5 phrases generated
  ✓ Failure modes: documented
  ✓ Examples:   3 included

  → Saved to ~/.claude/skills/tailwind-deduper/SKILL.md
  → Lint passes. Ready to use.
```

The next time you ask Claude "find duplicate Tailwind classes," your new skill activates.

---

## 🎯 How it works

1. **Intent extraction** — parses your description into structured fields
2. **Trigger generation** — produces 5–8 unambiguous phrases that should activate it
3. **Template instantiation** — fills the SKILL.md scaffold
4. **Self-lint** — runs the skill through `claude-code-skills-factory lint` before saving
5. **Optional preview** — opens the generated SKILL.md in your editor for review

---

## 📦 Commands

```bash
claude-skill new "<description>"          # Generate a new skill
claude-skill list                          # List your installed skills
claude-skill edit <name>                   # Re-generate with edits
claude-skill lint <path>                   # Validate any SKILL.md
claude-skill publish <name>                # Push to claude-skills-mega registry
claude-skill import <url>                  # Import from a URL or git repo
```

## 🔧 Advanced — bring your own templates

Drop custom templates in `~/.claude-skill-factory/templates/`:

```
templates/
├── default.md         # standard skill
├── code-review.md     # specialized review skill
├── ci-action.md       # GitHub Action helper
└── data-transform.md  # ETL-style skill
```

```bash
claude-skill new "Slack message formatter" --template ci-action
```

## 🌍 Multi-language

Describe in any language — the skill instructions are always in English (Claude works best in English internally), but you can pass `--locale=ar` to make the user-facing docs Arabic.

```bash
claude-skill new "أداة تكتب اختبارات pytest من docstrings" --locale ar
```

---

## 🛡️ Quality Standards

Every generated skill must pass:
- ✅ Required frontmatter fields present
- ✅ At least 5 specific triggers (no generic ones like "do X")
- ✅ "When NOT to use" section
- ✅ "Failure modes" section
- ✅ At least one concrete example
- ✅ No reference to non-existent tools/APIs

If generation can't meet the bar, it asks you for clarification instead of producing low-quality output.
